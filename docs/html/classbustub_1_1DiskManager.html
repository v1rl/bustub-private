Detailed Description

DiskManager takes care of the allocation and deallocation of pages within a database. It performs the reading and writing of pages to and from disk, providing a logical file layer within the context of a database management system.

DiskManager uses lazy allocation, meaning that it only allocates space on disk when it is first accessed. It maintains a mapping of page ids to their corresponding offsets in the database file. When a page is deleted, it is marked as free and can be reused by future allocations.
